● HTTP 400 的根因找到了。我来解释清楚然后修复。

  根因分析

  backend/app/middleware/sensitive.py 定义了一个敏感词中间件，在 main.py:263-274
   拦截所有 POST /api/v1/chat/* 请求，检测到敏感词就返回 HTTP 400。

  敏感词列表包含：诈骗、高利贷、赌博、毒品、洗钱、卖淫、走私、非法集资、暴力催收
  等。

  这是一个法律咨询机器人——用户必然问"我被诈骗了怎么办"、"高利贷可以不还吗"这类问
  题。关键词过滤直接拦截了这些合法法律咨询，返回 400。前端只看到 response.ok ===
   false，显示"抱歉，服务异常：Stream request failed"。

  时间线吻合：
  - 16:53:41 最后成功（问题不含敏感词）
  - 16:54:00 → 400（用户输入含法律敏感词的问题）
  - 16:54:13 → 400（再次尝试）

  修复

  1. 后端：敏感中间件跳过 /api/v1/chat 路径（法律咨询必须允许这些词汇）
  2. 前端：显示服务端真实错误信息，方便未来排查
